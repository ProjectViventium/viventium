# QA Templates

Use these templates when creating or refreshing a feature QA folder. Keep public reports sanitized and
preserve raw private evidence outside the public repo. New dated reports must include the full-view
evidence gate from `run-report.md`; if a required real user path was not run, mark the result
`BLOCKED` or `PARTIAL` instead of accepting substitute evidence. Confirm every public-safety item
and change `- [ ]` to `- [x]`; the v2 contract test rejects unchecked boxes.

- [`feature-readme.md`](feature-readme.md) - living feature QA scope and status
- [`cases.md`](cases.md) - durable case catalog
- [`run-report.md`](run-report.md) - dated execution report
