read-only inspection and supports explicitly confirmed email/calendar updates when the relevant
write tool is present. GlassHive brokered-worker acceptance remains separate under
`qa/glasshive-mcp-capability-broker/`.

## Forbidden outcomes (any of these is a FAIL)

- Asking "Gmail or Outlook?" for a generic inbox/email question when both providers are connected.
- Falling back to a GlassHive/browser/computer worker for a simple immediate read-only connected-account check when the Connected Accounts handoff is available and sufficient.
- Surfacing a forced download, `[local worker link]`, `View / Steer`, or other worker/run plumbing in the reply for a quick read.
- The main agent acquiring direct provider (`*_mcp_google_workspace` / `*_mcp_ms-365`) tools.
- Handing off for a request that is not about the user's own connected accounts (web/news, math, generation, code).
- Telling the user the Connected Accounts path only has read-only calendar/email access when a
  confirmed write-capable email/calendar tool is available.
- Mutating email/calendar content before explicit user confirmation, or using destructive/broad
  mail/calendar/file-write tools from the Connected Accounts handoff.
- A spurious completion-error bubble after a successful gpt-5.4 answer.
- Generic `Connection error` / `Something went wrong` / stale provider-rate-limit text when the real
  terminal blocker is a fallback connected-account reconnect requirement.
- Agent Builder > Advanced showing an unresolved handoff target as `Select agent` when the live main
  agent has a Connected Accounts handoff edge.

## Notes / residual

- A real confirmed send/draft or create/delete calendar smoke should be exercised only with explicit
  permission to mutate connected email/calendar data; the 2026-06-10 browser run proved the
  confirmation boundary, MS365 read reachability, and avoided creating live data.
- Handoff currently fans out to both providers for generic email/calendar; provider-narrowing ("only my gmail") is expected to be honored by the hand-off agent instructions but is not yet a separate case.
- CA-HANDOFF-008 durability gate: the auth fix was verified with the runtime relaunched from a keychain-attached shell (reparented to launchd, matching the helper's detached launch).
